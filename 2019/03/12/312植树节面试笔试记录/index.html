  1. String 的判等问题。代码：

    正确的输出结果是2 1。

    虽然我写对了，但是要分析为什么。

    今天查看了博客上有关“==”和“equals”的比较方式。

    ​ “==”判断符号左右两个对变量（引用类型）是否指向同一内存地址。而equals()方法判断两个对象是否一样（所有成员的值相同）。

    什么意思呢?String a = “10”;String b = “10”;那么a==b的值就是true。这是因为“10”是放在常量池中的，所以，引用a和b指向的都是常量池中的10的位置，当然是相等的。

    但是为什么String.valueOf()就不相等呢？这个要具体深入到String类的valueOf(int)方法中去了。

    getChars通过特殊的方法确定了使用char数组分别存储int类型的数值的个十百千万位上的准确存储。此时char数组buf的值变为[1,0]。再调用String的构造方法,这里最最最关键的部分，new方法new出了一个String，那么这个String就是被当做一个类，他不在是指向常量池，而是指向了具体的实例的内存，new总会产生新的对象。回过头看，String s = “10”又是经过怎样的操作。

    ​ 在JVM中存在一个字符串池，其中存有很对的String对象，并且可以被共享使用，所以当直接赋值String对象时，会首先在字符串常量池中查找是否存在相同的字符串定义，其判断依据是String对象中的equals方法的返回值。如果存在已经定义的话呢，就直接引用这个定义了。如果没有定义，则先创建对象加入到字符串池中，再返回引用。（卧槽，那这么说怎么着都是引用常量池的啊！难怪一定相等呢。）
